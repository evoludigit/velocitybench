**[Pattern] Anomaly Detection Patterns – Reference Guide**

---

### **Overview**
Anomaly Detection Patterns identify unusual or unexpected events, behaviors, or data points that deviate significantly from normal patterns. These patterns are widely used in security, fraud detection, predictive maintenance, and quality assurance systems. This guide outlines key concepts, implementation approaches, and practical examples for detecting anomalies in structured and unstructured data.

---

### **Key Concepts**
1. **Anomaly Definition**
   - Anomalies can be categorized as:
     - **Point anomalies** (individual data points deviating from the norm).
     - **Contextual anomalies** (data points abnormal in a specific context).
     - **Collective anomalies** (groups of data points behaving abnormally together).

2. **Detection Approaches**
   - **Statistical Methods:** Leverage statistical models (e.g., Z-scores, Gaussian Mixture Models) to model normality.
   - **Machine Learning:** Supervised (e.g., Isolation Forest, One-Class SVM) or unsupervised methods.
   - **Rule-Based:** Define explicit thresholds or thresholds based on domain rules.
   - **Deep Learning:** Useful for complex patterns (e.g., autoencoders, LSTM networks).
   - **Graph-Based:** Detect anomalies in network data (e.g., line graph structures).

3. **Metrics for Evaluation**
   - **Precision/Recall:** Measure accuracy in identifying true anomalies.
   - **False Positive/Fals Negative Rate:** Assess the trade-offs between accuracy and system load.
   - **F1 Score:** Balance between precision and recall.

## **Implementation Details**
### **1. Setting Up a Rule-Based Anomaly Detection System**
**Use Case:** Detecting transaction amounts exceeding a spending limit.

**Steps:**
1. **Define a Threshold:** Set a rule (e.g., `amount > $1,000`).
2. **Preprocess Data:** Clean transaction records for outliers and missing values.
3. **Query Example:**
   ```sql
   SELECT transaction_id, amount
   FROM transactions
   WHERE amount > 1000
   ORDER BY amount DESC
   LIMIT 100;
   ```
4. **Action:** Trigger an alert for flagged transactions.

---

### **2. Using Statistical Methods (Z-Scores)**
**Use Case:** Detecting sensor readings deviating from the mean.

**Steps:**
1. **Calculate Mean & Standard Deviation:**
   ```python
   import numpy as np
   readings = np.array([23, 25, 24, 120, 23, 25])
   mean = np.mean(readings)
   std_dev = np.std(readings)
   ```
2. **Compute Z-Scores:**
   ```python
   z_scores = [(x - mean) / std_dev for x in readings]
   ```
3. **Detect Anomalies:**
   ```python
   anomalies = [i for i, score in enumerate(z_scores) if abs(score) > 3]
   ```
4. **Output:**
   ```
   anomalies = [3]  # Index of the anomalous reading (120)
   ```

---

### **3. Using Isolation Forest (Supervised ML)**
**Use Case:** Detecting credit card fraud in transaction data.

**Steps:**
1. **Train the Model:**
   ```python
   from sklearn.ensemble import IsolationForest
   model = IsolationForest(contamination=0.01, random_state=42)
   model.fit(transaction_data)
   ```
2. **Predict Anomalies:**
   ```python
   anomalies = model.predict(transaction_data)
   anomalous_indices = np.where(anomalies == -1)[0]
   ```
3. **Query Example (Post-Prediction):**
   ```sql
   SELECT transaction_id, amount, user_id
   FROM transactions
   WHERE id IN (:anomalous_indices)
   ORDER BY amount DESC;
   ```

### **1. Simple SQL-Based Anomaly Detection**
```sql
-- Detect transactions exceeding average spending by 5x.
SELECT transaction_id, amount, user_id
FROM transactions
WHERE amount > (SELECT AVG(amount) * 5 FROM transactions)
ORDER BY amount DESC;
```

### **2. Detecting Anomalies in Time-Series Data**
```python
import pandas as pd
from statsmodels.tsa.seasonal import STL

# Load data
data = pd.read_csv("sensor_readings.csv", parse_dates=["timestamp"])

# Decompose into trend, seasonality, and residuals
stl = STL(data["reading"], period=24).fit()
residuals = stl.resid
anomalies = residuals[abs(residuals) > 3 * residuals.std()]

print(anomalies)
```

## **Related Patterns**
1. **[Threshold-Based Detection]**
   - Use explicit business rules (e.g., "flag transactions > $10k") to detect anomalies without ML.

2. **[Event Correlation]**
   - Combine anomaly detection with event correlation to identify multi-step attacks (e.g., two anomalies in sequence).

3. **[Graph-Based Anomaly Detection]**
   - Identify anomalies in network data (e.g., fraud rings in transaction graphs) using PageRank or community detection.

4. **[Feedback Loop for Continuous Learning]**
   - Refine anomaly models using labeled feedback (e.g., "mark this alert as false positive").

5. **[Real-Time Alerting]**
   - Integrate anomaly detection with alerting systems (e.g., Prometheus + Alertmanager).

---

### **Best Practices**
- **Baseline Establishment:** Train models on historical "normal" data.
- **Monitor Performance:** Regularly validate detection accuracy (e.g., precision/recall).
- **Dimensionality Reduction:** Use PCA or t-SNE for high-dimensional data.
- **Explainability:** Provide reasons for anomalies (e.g., "spending spike in new country").
- **Scalability:** Leverage distributed systems (e.g., Spark MLlib) for large datasets.
