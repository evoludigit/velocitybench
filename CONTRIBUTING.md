# Contributing to VelocityBench

Thank you for interest in contributing! This document provides guidelines for:
- Adding new frameworks
- Improving tests
- Reporting issues
- Code of conduct

## Adding a New Framework

See [docs/FRAMEWORK_SETUP.md](docs/FRAMEWORK_SETUP.md) for step-by-step instructions.

## Reporting Issues

Use GitHub Issues with:
- Clear title and description
- Framework name and version
- Steps to reproduce
- Expected vs actual behavior

## Code of Conduct

Be respectful, inclusive, and professional. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Testing

All changes must include:
- Unit tests (80%+ coverage)
- Integration tests
- Performance tests (if applicable)

See [TESTING_STANDARDS.md](../TESTING_STANDARDS.md).

---

Thank you for contributing to VelocityBench!
